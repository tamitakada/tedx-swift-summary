UIKit

UIKit is one framework used with Swift to build a user interface. It works using a system called auto layout, which is a way to position items by saying where each of their anchors are placed with respect to other items.

Each view has six anchors that it can be positioned with. Four of those anchors are the four sides, while two are the two center lines.

SwiftUI

SwiftUI is another UI Framework (like UIKit) released in 2019

Some issues with SwiftUI are…

  • It is newer than UIKit so there is less documentation
  • Doesn’t work on older computers & operating systems
  • Doesn’t include Storyboard

Some pros of SwiftUI are…

  • Very easy to see and edit your code
  • User friendly
  • Declarative Framework

Core Data

CoreData works with Swift to manage your app’s database through a Persistent Container Class. An instance of this Persistence Container Class allows access to three other classes that work with your apps data in different ways.

The three other classes are:

  • NSManagedObjectModel: an instance of this class describes your app’s types, properties and relationships
    • For a Fontbook, there is a class named "font". Each instance of the class "font" has the attributes name (which is font name) and isFavorited (which is whether something is favorited or not). This type of relationship is very easy to create with Core Data
    • For a Contact List, each person is a class and their information like name, number, and email is an attribute. With typical variables, you would make make a String for name... but this method of storing data does not preserve the relationship between people and the people's attributes. This is why Core Data is helpful!
  • NSManagedObjectContext: an instance of this class tracks changes to instances of your app’s types
  • NSPersistentStoreCoordinator saves and fetches instances of your app's type
